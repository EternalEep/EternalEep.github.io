Approach 1 - Hash Map

  • Hash Map
    • For self-balancing Binary Search Tree, search and insert are both $O(logn)$
    • For Hash Map, search and insert are both $O(1)$
  • Complexity Analysis
    • Time complexity: O(n)
    • Space complexity: O(n)
    • Runtime: 24ms

Approach 2 - Sorting

  • Sorting
    • If there are any duplicate integers, they will be consecutive after sorting.
  • Complexity Analysis
    • Time complexity: $O(nlogn)$ for heapsort
    • Space complexity: $O(1)$ for heapsort
    • Runtime: 28ms

Note

  • One should keep in mind that real world performance can be different from what the Big-O notation says. The Big-O notation only tells us that for sufficiently large input, one will be faster than the other.
  • Therefore, when nn is not sufficiently large, an $O(n)$ algorithm can be slower than an $O(nlogn)$ algorithm.

Leetcode287-Find the Duplicate Number

Posted on 2018-11-02 | Edited on 2018-11-15

Problem link

Solution

  • Problem note
  1. You must not modify the array (assume the array is read only).
  2. You must use only constant, O(1) extra space.
  3. Your runtime complexity should be less than $O(n^2)$.
  4. There is only one duplicate number in the array, but it could be repeated more than once.
  • Related Topics
    • CodeInterview3-数组中重复的数字
    • Leetcode217-Contains Duplicate

Approach 1- Modify the array (Not satisfied the requirments)

  • Referring to CodeInterview3-数组中重复的数字 and Leetcode217-Contains Duplicate
  • Complexity Analysis
    • Time complexity: $O(n)$
    • Space complexity: $O(1)$
    • Runtime: 4ms

Approach 2 - Sorting (Not satisfied the requirments)

  • Tips
    • If the nums are sorted, then any duplicate numbers will be adjacent in the sorted array.
  • Complexity Analysis
    • Time complexity: $O(nlogn)$
    • Space complexity: $O(n)/O(1)$
      • We sort nums in place, so the memory footprint is constant.
      • If we cannot modify the input array, then we must allocate linear space for a copy of nums and sort that insead.
    • Runtime: 8ms

Approach 3 - Set

  • Tips
    • Iterate over the array and insert each element into Set
  • Complexity Analysis
    • Time complexity: $O(n)$
      • Set rely on hash tables
    • Space complexity: $O(n)$
      • Set will contain n-1 distinct values
    • Runtime: 16ms

  • Tips
    • Split 1~n to 1~m and m+1~n.

Solution1 - Backtracking/Recursive method

  • Approach
    • Retrospective heuristic
    • Whether to choose the element to push into the set
    • Choose to push the element, select subsequent element to complete all trial for subsequent elements
    • Then pop back this element, select subsequent element to complete all trial for subsequent elements
  • Complexity Analysis
    • Time complexity: $O(2^n)$
    • Space complexity: $O(2^n)$
    • Runtime: 8ms

Solution2 - Bit Operation

  • Approach
    • 100=4 —-> A
    • 010=2 —-> B
    • 001=1 —-> C
    • 2^n categories
    • Construct some set, use A,B,C(int)&this set, if true, push into the set
  • Complexity Analysis
    • Time complexity: $O(n2^n)$
    • Space complexity: $O(2^n)$
    • Runtime: 8ms

Solution3 - No recursion

  • Approach
    • Current Set = Set in last layer + Current element (root is null)
  • Tips
    • back: choose the last element in vector (pop)
    • We should define size before loop, because when we push element into vector, it will be allocated new space, which will occur error!
  • Complexity Analysis
    • Time complexity: $O(2^n)$
    • Space complexity: $O(2^n)$
    • Runtime: 4ms

Solution

  • Approach 1
    • Search from matrix[0][0]
    • If target num < matrix[i][j]
    • Change the col index to j (Don’t need to search bottom right corner of the num)
  • Complexity Analysis
    • Time complexity: $O(m*n)$(worst)
    • Space complexity: $O(1)$
    • Runtime: 8ms
  • Approach 2
    • Search from num = matrix[0][col-1]
    • If target>num, row++
    • Else col–
  • Complexity Analysis
    • Time complexity:$O(m+n)$
    • Space complexity:$O(1)$
    • Runtime: 8ms
